# asset-lens 产品分析报告

## 一、执行摘要

asset-lens 是一款面向中国个人投资者的全栈式资产分析工具，通过 CLI + Web 双界面，覆盖从数据采集、盈亏计算、投资组合分析到 AI 辅助决策的完整投资管理闭环。产品在功能密度上已达到准机构级别，集成 10+ 数据源、20+ 子系统、深度整合机器学习与策略回测引擎，其 **A 股市场原生支持**和 **ML/AI 一体化能力**在全球开源竞品中独一无二。然而，当前产品面临严重的"功能密度与用户体验脱节"问题——强大的能力被高使用门槛（依赖 Python/CLI）和不成熟的 Web UI 所掩盖，导致潜在市场（百万级中国技术投资者）与实际用户（< 100 人）之间存在巨大鸿沟。**核心建议：立即将资源聚焦于"打通用户体验最后一公里"，优先完成 Web UI 核心功能重构、Docker 一键部署和券商数据自动导入器，而非继续堆砌功能。**

## 二、产品深度解析

### 2.1 目标用户与核心痛点

| 用户类型 | 特征 | 核心诉求 | 当前解决方案 |
|---------|------|---------|-------------|
| **量化爱好者 / Python 投资者** | 有编程背景，习惯用代码管理投资 | 自动化数据采集、盈亏计算、策略回测 | Excel + 手动脚本 |
| **多市场多资产投资者** | 持有 A 股、港股、美股、基金、加密货币 | 统一视图查看全局资产分布与真实收益率 | 各平台分别查看 |
| **数据隐私优先者** | 拒绝将资产上传至第三方理财 App | 本地化、自托管的解决方案 | 放弃使用数字工具 |
| **策略交易者** | 有系统化选股/择时策略 | 策略回测引擎、选股筛选器、机器学习预测 | 自建回测框架 |

### 2.2 核心功能矩阵

| 能力维度 | 核心模块 | 功能说明 | 成熟度 |
|---------|---------|---------|-------|
| **数据采集** | `data/` — 47 文件 | 集成 tushare、akshare、baostock、yfinance、Alpha Vantage、Finnhub、CCXT 等 **10+ 数据源**，覆盖 A 股/港股/美股行情、基本面、资金流 | ⭐⭐⭐⭐⭐ |
| **核心计算** | `core/` — 21 文件 | 多算法 IRR 计算、每日盈亏估算、实时 P&L、投资组合分析、DCA 定投解析 | ⭐⭐⭐⭐⭐ |
| **投资组合分析** | `analysis/` — 19 文件 | 组合仪表盘、再平衡建议、黑天鹅监测、盘前分析、预警监控 | ⭐⭐⭐⭐⭐ |
| **策略系统** | `strategy/` — 10 文件 | 选股筛选 (Screener)、策略引擎、回测器、组合评估、AI 选股分析 | ⭐⭐⭐⭐ |
| **机器学习** | `ml/` — 18 文件 | LightGBM/XGBoost 模型训练、预测、AI Trader、行业轮动模型、技术指标工程 | ⭐⭐⭐⭐⭐ |
| **报告与可视化** | `report/` — 23 文件 | HTML/PDF 报告、图表生成、日报/周报、模板引擎 | ⭐⭐⭐⭐ |
| **Web 界面** | `web/` — 10 文件 | FastAPI + React 前端，实时仪表盘 (WebSocket) | ⭐⭐⭐ |
| **AI 集成** | `ai/` — 4 文件 | LLM 提示构建器、AI 问答、LLM 驱动分析 (支持 OpenAI/LiteLLM/LangChain) | ⭐⭐⭐⭐ |

### 2.3 用户体验旅程分析

**理想用户旅程：**
1. **数据导入** → 自动拉取多市场行情 → 无需手动录入
2. **每日清算** → `make daily` 自动更新持仓、计算当日盈亏
3. **深度分析** → `make analyze` 查看持仓分布、收益曲线、风险暴露
4. **策略研究** → 选股筛选 → 回测验证 → ML 预测
5. **AI 辅助** → 自然语言问答获取投资建议
6. **报告输出** → 生成 HTML/PDF 日报/周报

**实际用户体验痛点：**
- **安装门槛高**：需要 Python 3.9+、conda/pip 基础，非技术用户无法上手
- **CLI 是主要交互方式**：39 个子命令需要记忆，缺乏可视化引导
- **Web UI 不成熟**：远未达到 Ghostfolio 级别的用户体验
- **数据导入依赖手动 CSV**：缺乏对主流券商（东方财富、富途）导出文件的直接解析
- **文档不足**：缺少完整用户手册、场景化教程和 API 文档

### 2.4 技术架构亮点

1. **数据源冗余设计**：`data/providers/registry.py` 实现多 provider 注册和 fallback 机制，单一数据源不可用时自动切换——这在个人投资工具中是专业的架构决策
2. **模块化 CLI 体系**：采用 click 框架 + rich 输出，39 个 CLI 模块文件构成清晰的命令树
3. **多模态输出**：CLI 格式化输出 + HTML 报告 + PDF 输出 + Web Dashboard + AI 自然语言分析
4. **深度 ML 集成**：从特征工程 → 模型训练 → 预测 → 回测 → 实盘模拟的完整闭环

## 三、竞争分析与市场定位

### 3.1 主要竞品对比

| 维度 | **asset-lens** | **Ghostfolio** | **Portfolio Performance** | **OpenBB** | **Beancount/Fava** |
|------|---------------|---------------|--------------------------|-----------|---------------------|
| **技术栈** | Python CLI + Web | Angular + PostgreSQL | Java 桌面 | Python CLI + SDK | Python CLI |
| **数据源** | **10+（A股原生支持）** | Yahoo Finance为主 | 手动+Yahoo | 多源（美股为主） | 手动记账 |
| **A 股支持** | ⭐⭐⭐⭐⭐ **原生支持** | ❌ 不支持 | ❌ 不支持 | ❌ 不支持 | 需手动配置 |
| **ML/AI** | ⭐⭐⭐⭐⭐ LightGBM/XGBoost/LLM | ❌ 无 | ❌ 无 | ⭐⭐⭐ 基础 | ❌ 无 |
| **策略回测** | ⭐⭐⭐⭐ 有回测引擎 | ❌ 无 | ❌ 无 | ⭐⭐⭐ 有 | ❌ 无 |
| **IRR 计算** | ⭐⭐⭐⭐⭐ 多种算法 | ⭐⭐⭐ 基础 | ⭐⭐⭐⭐ 较好 | ⭐⭐⭐ 基础 | ⭐⭐⭐ 基础 |
| **Web UI** | ⭐⭐⭐ FastAPI+React | ⭐⭐⭐⭐⭐ 优秀 | ⭐⭐⭐ Java桌面 | ⭐⭐⭐⭐ 较好 | ⭐⭐⭐⭐ Fava |
| **部署体验** | 需 Python 环境 | Docker 一键部署 | Java 桌面安装 | pip install | Python |
| **社区规模** | 个人项目 | 6k+ Stars | 中等 | 30k+ Stars | 15k+ Stars |

### 3.2 差异化优势（核心竞争力）

1. **A 股市场原生支持 —— 唯一性壁垒**
   - 全球主要开源投资工具中，asset-lens 是唯一将 A 股作为原生市场的产品
   - 原生集成 tushare、akshare、baostock 等 A 股数据源，理解 T+1、涨跌停、分红送股等交易规则
   - 这是任何全球性工具（Ghostfolio、OpenBB）无法短期内复制的差异化优势

2. **ML/AI 一体化能力 —— 技术护城河**
   - 从技术指标工程 → LightGBM/XGBoost 训练 → 预测 → 回测 → AI Trader，形成完整闭环
   - Ghostfolio 和 Portfolio Performance 在 ML 领域几乎空白
   - LLM 驱动的 AI 分析功能，支持自然语言问答

3. **功能密度极高 —— 一站式平台**
   - 单个项目同时包含 IRR 计算、组合分析、策略回测、ML 预测、风险监控、通知调度
   - 功能广度在同类竞品中无出其右
   - 用户无需在多个工具间切换

### 3.3 市场定位与竞争格局

**核心定位：面向中国量化投资者及数据敏感型投资者的"本地优先的一站式投资分析与决策助手"**

**竞品格局中的位置：**
- **全球市场**：asset-lens 的 A 股专注度和 ML 能力远超 Ghostfolio/Portfolio Performance，但 UI/UX 和社区规模差距显著
- **中国市场**：在开源领域几乎没有直接竞品——中国个人投资工具市场存在明显的蓝海空白
- **替代品竞争**：主要竞争对手不是其他开源项目，而是 Excel、且慢/雪球组合、以及同花顺/东方财富 App

**市场空白：**
中国 A 股个人投资者约 2.2 亿，其中量化/技术派投资者约 1100-2200 万，有 Python 编程能力者约 50-100 万。asset-lens 当前月活预计 < 100 人，**增长空间巨大**。

## 四、战略建议

### 4.1 产品定位调整

**现状问题：** 当前定位"个人资产操作系统"过于宽泛，未清晰传达核心价值。产品"身份认知"模糊，同时面向量化爱好者、多资产投资者、隐私党、策略交易者，导致市场沟通失焦。

**建议调整：**
- **核心关键词：** 从"操作系统"转向"分析与决策助手"
- **首要用户群：** 明确为"有能力、有意愿使用代码的量化投资者"
- **核心卖点：** 强化"本地优先、隐私安全、可定制"的差异化价值
- **一句话定位升级：** "**asset-lens：你的本地化投资大脑，让你像机构一样分析你的资产**"

### 4.2 增长策略

**核心获客渠道（按优先级排序）：**

1. **内容营销（最高优先级、最低成本）**
   - 在知乎、V2EX、即刻等社区发布深度教程
   - 内容主题：《用一条 Python 命令管理你 10 个证券账户的持仓》《手把手教你用开源工具复现基金经理的选股策略》
   - 输出一次、长期获客，ROI 极高

2. **开源社区运营**
   - 从"个人项目"转变为"社区项目"
   - 在 GitHub 上开放 Issue 作为需求征集和用户反馈中心
   - 响应每一个 Issue，引导用户参与改进

3. **KOL/技术达人合作**
   - 寻找量化投资、数据可视化领域的影响力人物
   - 提供深度使用体验，换取评测或教程内容

**社区运营策略：**
- 建立"资产配置模板库"，允许用户分享脱敏后的配置方案
- 举办"最佳策略回测比赛"，获胜策略成为官方示例

### 4.3 产品路线图优先级

**下一阶段（6 个月）核心目标：打通用户体验的最后一公里**

**必须做（高优先级）：**

| 任务 | 时间线 | 影响力 | 成本 | 说明 |
|-----|-------|-------|-----|------|
| **Web UI 核心功能重构** | 1-3 个月 | 极高 | 中 | 将 `daily`, `analyze`, `report` 三个核心 CLI 命令以优雅的 Web 仪表盘呈现 |
| **Docker 一键部署** | 1 个月 | 高 | 低 | 提供 `docker-compose.yml`，用户只需 `docker compose up` |
| **券商数据自动导入器** | 2-3 个月 | 极高 | 中 | 解析东方财富、同花顺、富途的导出文件，解决最大录入痛点 |
| **AI 助手 Chat UI** | 2-3 个月 | 中 | 低 | 在 Web 上提供 AI 投资助手聊天气泡式交互 |
| **内容营销启动** | 立即 | 极高 | 极低 | 撰写中文 README，发布知乎教程 |

**不应该做（延后处理）：**
- ❌ 移动端 App / 小程序（在未证明 PMF 前是资源浪费）
- ❌ 多用户/团队协作（偏离个人工具定位）
- ❌ 高级 ML 模型优化（当前模型已足够，ROI 低）
- ❌ 税务分析计算（复杂且高度本地化）

### 4.4 风险与缓解策略

| 风险 | 描述 | 缓解策略 |
|-----|------|---------|
| **市场太小** | 目标用户群可能只是小众，难以支撑增长 | 通过 Web UI 和券商导入器将用户扩大到"懂技术的投资者" |
| **巨头入场** | 东方财富/同花顺推出类似功能 | 坚守"本地优先、隐私安全、可定制"的核心价值观 |
| **开发者 burnout** | 个人项目维护压力大 | 开放社区贡献，及时响应 Issue/PR，吸引贡献者 |
| **数据源依赖** | tushare/akshare 等免费数据源不稳定 | 利用 registry fallback 机制，文档说明付费数据源配置 |

## 五、结论

**核心判断：asset-lens 是中国个人投资开源领域最具潜力的产品，当前处于"功能完工但体验未达"的关键拐点，值得全力投入。**

**投资建议：**
- ✅ **值得投入** — 技术护城河扎实，市场空白明确，增长空间巨大
- ✅ **建议立即启动** — 优先完成 Web UI 核心功能 + Docker 部署 + 内容营销三板斧

**下一步行动清单：**

1. **立即开始**
   - 重写中文 README，清晰传达核心价值
   - 在知乎发布第一篇深度教程
   - 开放 GitHub Issue 作为反馈渠道

2. **1-2 周内**
   - 完成 Docker 部署方案
   - 确定券商数据导入器的开发计划

3. **1-3 个月内**
   - Web UI 核心功能 MVP 上线
   - 发布券商数据导入器第一个版本
   - 积累首批 100 名活跃用户

4. **3-6 个月内**
   - Web UI 达到 Ghostfolio 级别的基础体验
   - 社区贡献者达到 3-5 人
   - 月活用户突破 1000 人

asset-lens 不缺能力，缺的是"让能力被看见"的最后一公里。**下一步的战略重心不是堆砌功能，而是将产品从"开发者的瑞士军刀"升级为"投资者的作战指挥中心"。**
